Agile/Scrum is only a fit for mature software systems. It should not be used for the initial development of the system.

It creates total chaos to try and plan and divide work for a new project where a system is being built from scratch and requirements are ambiguous and there are many unknowns. It's best to have 1 developer heads-down work on it in an unstructured fashion for many months until there's a solid foundat
